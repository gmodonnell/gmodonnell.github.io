SSH Keys for GitHub

I am writing this because every time I get a new laptop I always forget this process and it's easier to write it down in a single, concise manner rather than reading through the two necessary GitHub articles that appear. 


ssh-keygen -t ed25519 -C "email@email.dom"

eval "$(ssh-agent -s)"

ssh-add ~/.ssh/[keyfile]

This is how you make and add the key on your device. Then you add your public key on the website. When you clone repos, you have to clone them using ssh in the following format in order to be able to commit using ssh. 


git clone git@github.com:/[author]/[repo]

If you mess up and need cloned a repo over https, pushing to the repo will cause you to try to do it over https which is deprecated so you will have to set the repo push destination to ssh using the following command 


git remote set-url [remote name] [remote url]

The [remote name] is usually origin or main, and the [remote url] is in the format git@github.com/[author]/[repo].
